# 验证清单

Landing page 改造完成后逐项检查。

## 结构

- [ ] Section 顺序符合"痛点 → 解决方案 → 证明 → 行动"叙事弧线
- [ ] Hero 后紧跟信任信号，不是直接跳到功能展示
- [ ] 社会证明放在用户产生疑虑的节点（How It Works 之后），不是页面底部
- [ ] CTA 出现在至少 2 个位置（Hero + 表单区/页面底部）

## 首屏

- [ ] 标题是收益导向的（不是功能描述）
- [ ] 副标题补充具体怎么实现
- [ ] CTA 旁边有安全感微文案
- [ ] 版本号、badge 或其他信用信号可见
- [ ] 5 秒内能回答"为谁带来什么结果"

## 信任信号

- [ ] 3 个信任信号在 Hero 下方
- [ ] 每个信号图标 + 标题 + 一句话描述
- [ ] 每个信号直接打消一个用户顾虑

## How It Works

- [ ] 3 个步骤（不超过 4 个）
- [ ] 每个步骤有编号、图标、标题、描述
- [ ] 描述是一句话，不是段落
- [ ] 步骤之间有视觉连线或序号

## Benefits

- [ ] 6 个左右卡片（4-8 均可）
- [ ] 每个卡片是痛点 → 解决方案的结构
- [ ] 图标 + 标题 + 描述
- [ ] 描述具体、量化，不是空泛承诺

## 安全感

- [ ] 每个 CTA 按钮附近有降低风险的微文案
- [ ] 表单上方有信任信号条（✓ 开头）
- [ ] Pricing 区域 CTA 文案简洁直接

## 文案

- [ ] 没有"赋能/打造/闭环"等禁用词
- [ ] 标题没有用动词祈使句催用户
- [ ] CTA 是具体动词，不是"了解更多"等无信息量文案
- [ ] 没有承诺收益
- [ ] 没有"不是 xxx，而是 xxx"的结构

## 视觉

- [ ] How It Works 步骤有可视化（编号/图标/连线）
- [ ] Benefits 卡片有图标
- [ ] 不是纯文字墙
- [ ] 移动端 1 列、平板 2 列、桌面 3 列

## 技术

- [ ] `pnpm build` 无错误
- [ ] 所有语言版本正常渲染
- [ ] Dark mode 样式正确
- [ ] 动画正常触发
- [ ] CTA 链接正确
- [ ] 翻译键无遗漏
